Ini penjelasannya: 📁 JSON vs 🗄️ MySQL — Apa Bedanya? Di proyek KTI kamu, data disimpan di transactions.json Artinya tidak pakai MySQL sama sekali. JSON hanyalah file teks biasa yang berisi data. 🤔 Lalu apa fungsi MySQL? MySQL adalah database sungguhan yang digunakan jika proyek kamu lebih besar/profesional. Bedanya: Fitur transactions.json MySQL Penyimpanan File .json di folder Server database terpisah Kecepatan Lambat jika data banyak Cepat meski jutaan data Keamanan Mudah diedit manual Terlindungi dengan password Pencarian Harus baca semua data Bisa filter/query spesifik Multi-user Bisa konflik/bentrok Aman diakses banyak user Cocok untuk Proyek kecil / KTI Aplikasi nyata / production 📦 Isi MySQL (jika pakai) Kalau pakai MySQL, kamu akan punya tabel seperti ini: -- Nama database CREATE DATABASE kas_digital; -- Tabel transaksi (pengganti transactions.json) CREATE TABLE transactions ( id INT AUTO_INCREMENT PRIMARY KEY, tanggal DATE NOT NULL, keterangan VARCHAR(255) NOT NULL, jenis ENUM('pemasukan', 'pengeluaran') NOT NULL, jumlah BIGINT NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); ✅ Kesimpulan untuk KTI kamu Proyek kamu sudah benar menggunakan transactions.json — untuk skala KTI/siswa itu cukup dan valid. MySQL baru diperlukan kalau aplikasinya dipakai banyak orang secara bersamaan atau datanya sangat besar.
